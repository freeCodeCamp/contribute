# Index

## CSS Helper

The CSS Helper is used to parse and test CSS written by campers.

- [getStyle()](#getstyle) – Returns a CSS style declaration for a given selector.
- [getStyleAny()](#getstyleany) – Returns the first matching CSS style declaration from multiple selectors.
- [getCSSRules()](#getcssrules) – Retrieves an array of CSS rules for a specified at-rule (e.g., media, fontface).
- [getRuleListsWithinMedia()](#getrulelistswithinmedia) – Returns the CSS rules inside a specific media query.
- **Less Frequent Methods**
  - [getStyleDeclarations()](#getstyledeclarations) – Returns an array of style declarations for a selector.
  - [isPropertyUsed()](#ispropertyused) – Checks if a specific CSS property is used anywhere in the code.
  - [getStyleRule()](#getstylerule) – Returns a style rule with additional methods (e.g., isDeclaredAfter).
  - [getStyleSheet()](#getstylesheet) – Returns the entire parsed CSS as a style sheet object.
  - [selectorsFromSelector()](#selectorsfromselector) – Provides alternate selectors based on a given one.

## JS Helper

The JS Helper provides utilities to test and manipulate JavaScript code.

- **Mockers**
  - [RandomMocker](#randommocker) – Mocks Math.random to generate a predictable sequence.
  - [spyOn](#spyon) – Wraps a method to record its calls and return values.
- **Static Methods**
  - [isCalledWithNoArgs()](#iscalledwithnoargs) – Checks if a function is called without any arguments.
  - [concatRegex()](#concatregex) – Combines two regexes or strings into a single regex.
  - [functionRegex()](#functionregex) – Creates a regex to match a function declaration with specified parameters.
- [Strip Content](#strip-content) – Provides methods to remove comments and whitespace from code.
- [permutateRegex](#permutateregex) – Generates a regex that matches elements in any order.

# CSS Helper

The CSS Helper assists with testing campers' CSS code. You instantiate the helper within a test block as follows:

<details>
  <summary>Example Usage</summary>

```js
const helper = new __helpers.CSSHelp(document);
```

</details>

## getStyle()

This method takes a CSS selector and returns a CSS style declaration object. For example, if the camper has written:

<details>
  <summary>Example Usage</summary>

```css
body {
  background: linear-gradient(45deg, rgb(118, 201, 255), rgb(247, 255, 222));
  margin: 0;
  padding: 0;
  width: 100%;
  height: 100vh;
  overflow: hidden;
}
```

</details>

Then calling:

<details>
  <summary>Example Usage</summary>

```js
assert.strictEqual(helper.getStyle('body')?.width, '100%');
```

</details>

The returned object contains a native method `getPropertyValue` as well as a helper method `getPropVal()`.

## getStyleAny()

This method takes multiple CSS selectors and returns the first matching CSS style declaration object. For example, if the camper has written:

<details>
  <summary>Example Usage</summary>

```css
.sky {
  background: radial-gradient(
    closest-corner circle at 15% 15%,
    #ffcf33,
    #ffcf33 20%,
    #ffff66 21%,
    #bbeeff 100%
  );
}
```

</details>

Then calling:

<details>
  <summary>Example Usage</summary>

```js
assert.strictEqual(helper.getStyleAny(['.earth', '.sky'])?.width, '100%');
```

</details>

Similarly, the returned object provides a `getPropVal()` method.

## getCSSRules()

This method takes an at-rule type from the union `media | fontface | import | keyframes` and returns an array of CSS rules matching that at-rule. For example, if the camper has written:

<details>
  <summary>Example Usage</summary>

```css
@media (max-width: 100px) {
  body {
    background-color: green;
  }
}
```

</details>

Then:

<details>
  <summary>Example Usage</summary>

```js
const rules = helper.getCSSRules('media');
```

</details>

The returned array includes objects representing each matching media rule.

## getRuleListsWithinMedia()

This method takes a media query text (e.g. `("max-width: 200")`) and returns the CSS rules within that media query. It is equivalent to accessing the media rule's `cssRules` property.

---

## Less Frequent Methods

### getStyleDeclarations()

Takes a CSS selector and returns an array of CSS style declaration objects.

### isPropertyUsed()

Checks if a specific CSS property is used anywhere in the camper's CSS.

### getStyleRule()

Returns the CSS style declaration for a given selector, with an extra method `isDeclaredAfter()`.

### getStyleSheet()

Returns the camper's CSS parsed into a CSS Style Sheet object.

### selectorsFromSelector()

Returns alternate selectors based on a given selector (e.g., `a`, `label > a`, etc.).

# JS Helper

The JS Helper provides methods to test and manipulate JavaScript code.

## Mockers

### RandomMocker

Mocks `Math.random` for testing purposes, ensuring a predictable sequence of random numbers.

<details>
  <summary>Example Usage</summary>

```js
const randomMocker = new RandomMocker();
randomMocker.mock();
Math.random(); // first call is always 0.2523451747838408
Math.random(); // second call is always 0.08812504541128874
randomMocker.mock();
Math.random(); // resets to 0.2523451747838408
randomMocker.restore();
Math.random(); // native Math.random is restored
```

</details>

### spyOn

Wraps an object's method to record its calls and return values.

<details>
  <summary>Example Usage</summary>

```js
const obj = { method: (arg1, arg2) => `called by: ${arg1} + ${arg2}` };
const spy = helper.spyOn(obj, 'method');
obj.method('arg', 'another arg');
obj.method('2nd call');
spy.calls; // [["arg", "another arg"], ["2nd call"]]
spy.returns; // ["called by: arg + another arg", "called by: 2nd call + undefined"]
// Restore the original method:
spy.restore();
```

</details>

## Static Methods

### isCalledWithNoArgs()

Checks if a function is invoked without any arguments, ignoring commented-out calls.

### concatRegex()

Combines two regexes or strings into a single compiled regex.

<details>
  <summary>Example Usage</summary>

```js
const regex1 = /a\s/;
const regex2 = /b/;
concatRegex(regex1, regex2).source === 'a\\sb';
```

</details>

### functionRegex()

Creates a regex to match a function declaration given its name and parameters. Options include capturing the function body.

<details>
  <summary>Example Usage</summary>

```js
const regex = functionRegex('foo', ['bar', 'baz']);
regex.test('function foo(bar, baz){}'); // true
regex.test('function foo(bar, baz, qux){}'); // false
regex.test('foo = (bar, baz) => {}'); // true
```

</details>

Additional examples with options:

<details>
  <summary>Example Usage (capture option)</summary>

```js
const regEx = functionRegex('foo', ['bar', 'baz'], { capture: true });
const combinedRegEx = concatRegex(/var x = "y"; /, regEx);

const match = `var x = "y";
function foo(bar, baz){}`.match(regex);
match[1]; // "function foo(bar, baz){}"
```

</details>

<details>
  <summary>Example Usage (includeBody option)</summary>

```js
const regEx = functionRegex('foo', ['bar', 'baz'], { includeBody: false });
const match = `function foo(bar, baz){console.log('ignored')}`.match(regex);
match[1]; // "function foo(bar, baz){"
```

</details>

Note: The capture option may not work as expected with arrow functions.

## Strip Content

Static methods available:

- `__helpers.removeCssComments()`
- `__helpers.removeHtmlComments()`
- `__helpers.removeJSComments()`
- `__helpers.removeWhitespace()`

These methods remove comments and whitespace from the camper's code.

## permutateRegex

Generates a regex that matches elements in any order. It accepts options to customize separators between elements and permutations.

<details>
  <summary>Example Usage</summary>

```js
const source1 = 'a';
const regex1 = /b/;
const source2 = 'c';

permutateRegex([source1, regex1, source2]).source ===
  new RegExp(
    /(?:a\s*\|\|\s*b\s*\|\|\s*c|b\s*\|\|\s*a\s*\|\|\s*c|c\s*\|\|\s*a\s*\|\|\s*b|a\s*\|\|\s*c\s*\|\|\s*b|b\s*\|\|\s*c\s*\|\|\s*a|c\s*\|\|\s*b\s*\|\|\s*a)/
  ).source;
```

</details>

Other examples with custom separators:

<details>
  <summary>Example Usage (elementsSeparator)</summary>

```js
permutateRegex(['a', /b/, 'c'], { elementsSeparator: ',' }).source ===
  new RegExp(/(?:a,b,c|b,a,c|c,a,b|a,c,b|b,c,a|c,b,a)/).source;
```

</details>

<details>
  <summary>Example Usage (permutationsSeparator)</summary>

```js
permutateRegex(['a', /b/, 'c'], { permutationsSeparator: '&' }).source ===
  new RegExp(
    /(?:a\s*\|\|\s*b\s*\|\|\s*c&b\s*\|\|\s*a\s*\|\|\s*c&c\s*\|\|\s*a\s*\|\|\s*b&a\s*\|\|\s*c\s*\|\|\s*b&b\s*\|\|\s*c\s*\|\|\s*a&c\s*\|\|\s*b\s*\|\|\s*a)/
  ).source;
```

</details>

# AST-based Helpers (Python)

AST-based Helpers allow you to analyze Python code by converting it into an Abstract Syntax Tree (AST). They are run in Python within tests like so:

<details>
  <summary>Example Usage</summary>

```js
({ test: () => assert(runPython(`<python code>`)) });
```

</details>

The primary class is `_Node`, corresponding to `ast_helpers.Node` in Python. Once instantiated with `_Node(_code)`, you can chain methods to explore the AST.

## find_function()

Searches for a function by name and returns its AST node.

<details>
  <summary>Example Usage</summary>

```py
_Node('def foo():\n  x = "1"').find_function("foo").has_variable("x")  # True
```

</details>

## find_functions()

Returns an array of nodes for functions with the same name.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
class Spam(ABC):
  @property
  @abstractmethod
  def foo(self):
    return self.x

  @foo.setter
  @abstractmethod
  def foo(self, new_x):
    self.x = new_x
"""
explorer = Node(code_str)
len(explorer.find_class("Spam").find_functions("foo"))  # 2
```

</details>

## find_async_function()

Finds an asynchronous function.

<details>
  <summary>Example Usage</summary>

```py
_Node('async def foo():\n  await bar()').find_async_function("foo").is_equivalent("async def foo():\n  await bar()")  # True
```

</details>

## find_awaits()

Retrieves all await expressions within an async function.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
async def foo(spam):
  if spam:
    await spam()
  await bar()
  await func()
"""
explorer = Node(code_str)
explorer.find_async_function("foo").find_awaits()[0].is_equivalent("await bar()")  # True
```

</details>

## find_variable()

Searches for variable assignments in the code.

<details>
  <summary>Example Usage</summary>

```py
_Node("y = 2\nx = 1").find_variable("x").is_equivalent("x = 1")
```

</details>

Also supports type annotations and object properties:

<details>
  <summary>Example Usage (annotations)</summary>

```py
_Node("a: int = 1").find_variable("a").is_equivalent("a: int = 1")
```

</details>

<details>
  <summary>Example Usage (object properties)</summary>

```py
_Node("self.spam = spam").find_variable("self.spam").is_equivalent("self.spam = spam")
```

</details>

## find_aug_variable()

Finds augmented assignments (e.g., x += 1).

<details>
  <summary>Example Usage</summary>

```py
_Node("x += 1").find_aug_variable("x").is_equivalent("x += 1")
_Node("x -= 1").find_aug_variable("x").is_equivalent("x -= 1")
```

</details>

## find_body()

Returns the body of a function as an AST node.

<details>
  <summary>Example Usage</summary>

```py
func_str = """
def foo():
    x = 1
"""
Node(func_str).find_function("foo").find_body().is_equivalent("x = 1")  # True
```

</details>

## find_return()

Finds return statements in a function.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
def foo():
  if x == 1:
    return False
  return True
"""
explorer = Node(code_str)
explorer.find_function("foo").find_return().is_equivalent("return True")  # True
```

</details>

It can also find returns inside if blocks:

<details>
  <summary>Example Usage</summary>

```py
explorer.find_function("foo").find_ifs()[0].find_return().is_equivalent("return False")  # True
```

</details>

## find_calls()

Searches for function calls.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
print(1)
print(2)
foo("spam")
obj.foo("spam")
obj.bar.foo("spam")
"""
explorer = Node(code_str)
len(explorer.find_calls("print"))  # 2
```

</details>

And inspects individual calls:

<details>
  <summary>Example Usage</summary>

```py
explorer.find_calls("print")[0].is_equivalent("print(1)")
explorer.find_calls("print")[1].is_equivalent("print(2)")
```

</details>

## find_call_args()

Retrieves arguments from a function call.

<details>
  <summary>Example Usage</summary>

```py
explorer = Node("print(1, 2)")
len(explorer.find_calls("print")[0].find_call_args())  # 2
```

</details>

## find_class()

Searches for a class definition.

<details>
  <summary>Example Usage</summary>

```py
class_str = """
class Foo:
  def __init__(self):
    pass
"""
Node(class_str).find_class("Foo").has_function("__init__")  # True
```

</details>

## find_ifs()

Finds all if statements in the code.

<details>
  <summary>Example Usage</summary>

```py
if_str = """
if x == 1:
  x += 1
elif x == 2:
  pass
else:
  return

if True:
  pass
"""
Node(if_str).find_ifs()[0].is_equivalent("if x == 1:\n  x += 1\nelif x == 2:\n  pass\nelse:\n  return")
```

</details>

<details>

<details>

## find_for_loops()

Finds all for loops.

<details>
  <summary>Example Usage</summary>

```py
for_str = """
dict = {'a': 1, 'b': 2, 'c': 3}
for x, y in enumerate(dict):
    print(x, y)
else:
    pass

for i in range(4):
    pass
"""

```

</details>

## find_for()

Finds a for loop matching specific variables and iterators.

<details>
  <summary>Example Usage</summary>

```py

explorer.find_for("i", "range(4)").is_equivalent("for i in range(4):\n  pass")
```

</details>

## find_for_vars()

<details>
  <summary>Example Usage</summary>

```py
explorer.find_for_loops()[1].find_for_vars().is_equivalent("i")
```

</details>

## find_for_iter()

Returns the iterable part of a for loop.

<details>
  <summary>Example Usage</summary>

</details>

## find_bodies()

Gets the bodies of control structures like if, while, or for loops.

<details>
  <summary>Example Usage</summary>

```py
if_str = """
if True:
  x = 1
elif False:
  x = 2
"""

```

</details>

## find_imports()

Finds import statements.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
import ast, sys
from math import factorial as f
"""
explorer = Node(code_str)
len(explorer.find_imports())  # 2
```

</details>

## find_comps()

Retrieves list comprehensions, set comprehensions, dict comprehensions, and generator expressions not assigned to a variable.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
[i**2 for i in lst]
(i for i in lst)
{i * j for i in spam for j in lst}
{k: v for k,v in dict}
comp = [i for i in lst]
"""
explorer = Node(code_str)
len(explorer.find_comps())  # 4
```

</details>

## find_comp_iters()

Returns the iterables used in comprehensions or generator expressions.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
x = [i**2 for i in lst]

def foo(spam):
  return [i * j for i in spam for j in lst]
"""
explorer = Node(code_str)
len(explorer.find_variable("x").find_comp_iters())  # 1
```

</details>

## find_comp_targets()

Returns the target variables in comprehensions.

<details>
  <summary>Example Usage</summary>


</details>

## find_comp_key()

Finds the key in a dictionary comprehension.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
```

</details>

## find_comp_expr()

Retrieves the expression evaluated in comprehensions, including any if/else constructs.

<details>
  <summary>Example Usage</summary>

```py
code_str = """

def foo(spam):
  return [i * j for i in spam for j in lst]
"""
explorer = Node(code_str)
explorer.find_variable("x").find_comp_expr().is_equivalent("i**2 if i else -1")
```

</details>

## find_comp_ifs()

Returns any if conditions used in comprehensions.

<details>
  <summary>Example Usage</summary>

```py
code_str = """
x = [i**2 if i else -1 for i in lst]

def foo(spam):
  return [i * j for i in spam if i > 0 for j in lst if j != 6]
"""
explorer = Node(code_str)
len(explorer.find_function("foo").find_return().find_comp_ifs())  # 2
```

</details>

## has_stmt()

Checks for the presence of a specific statement.
